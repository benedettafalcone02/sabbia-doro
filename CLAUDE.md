## Architecture

**Sabbia d'Oro** is a mobile-first beach management PWA for a single season. It tracks 292 beach spots (postazioni): 84 palme and 208 ombrelloni (split into Settore A / B).

### Navigation

There is **no React Router**. `App.jsx` holds a `page` string in `useState` and renders the matching page component conditionally. Navigation is done by calling `onNavigate(pageName)` passed as a prop. Pages: `dashboard`, `mappa`, `prenota`, `clienti`, `disponibilita`, `admin`.

### Data layer — `useStore` (`src/hooks/useStore.js`)

Single source of truth. On mount it fetches all rows from the Supabase `occupazioni` table and derives everything in memory:

- **postazioni** — the 292 spot definitions (generated by `generatePostazioni()` in `src/lib/data.js`) merged with occupancy data from Supabase.
- **clienti** — built by scanning unique `cliente` values across occupied rows; not a separate DB table.
- **prenotazioni** — virtual objects built from `occupazioni` rows for dashboard use; payment state is not persisted to the DB.

`salvaCliente` and `registraPagamento` are local-only (no DB write). Only `Prenota` and `ImportExcel` write to Supabase.

### Database

Single Supabase table: **`occupazioni`**. Multiple rows per `(tipo, numero)` are allowed — the unique constraint on that pair was dropped to support subaffitti and disponibilità sovrapposti alla prenotazione stagionale.

| column | type | notes |
|--------|------|-------|
| tipo | text | `'palma'` or `'ombrellone'` |
| numero | integer | spot number within its type |
| cliente | text | name stored in uppercase |
| tipo_occupazione | text | `'stagionale'`, `'subaffitto'`, `'subaffitto_disponibile'` |
| data_inizio / data_fine | date | mandatory; used to pick the active booking for today |
| lettini / sdraio / regista | integer | equipment counts |

`useStore` groups all rows by `(tipo, numero)`, picks the one active today (subaffitto/disponibile take priority over stagionale), and builds a `prenotazioni` array with all future bookings for the timeline view.

### Postazione numbering

- **Palme**: numbered 1–84 sequentially row by row (6 rows × 14 columns).
- **Ombrelloni**: numbered 1–208 in serpentine order starting from row 13 down to row 1 (13 rows × 16 columns). Odd steps reverse the column order. Settore A = rows 1–6, Settore B = rows 7–13.

This logic lives entirely in `generatePostazioni()` in `src/lib/data.js` and must stay consistent with DB data.

### Excel import (`src/components/ImportExcel.jsx`)

Detects file type by column header: `'Numero Palma'` or `'Numero Ombrellone'`. Parses the `Attrezzatura` column with a regex supporting codes like `2L`, `LR`, `LS`, `2LR`, `LRS` (L=lettino, S=sdraio, R=regista). On import it **deletes all rows for that tipo first**, then batch-inserts in chunks of 50.

### Styling

Global CSS classes (`btn`, `card`, `badge-*`, `form-group`, etc.) are defined in `src/styles/global.css`. CSS variables use `--navy`, `--yellow`, `--red`, `--sky`, `--green`. Component-specific styles use CSS Modules (e.g., `Mappa.module.css`). The app is mobile-first; breakpoint at 768px.

### Deployment

Deployed to Vercel. `vercel.json` is minimal. The `.vercel/` directory contains the linked project config.
